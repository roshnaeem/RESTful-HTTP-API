SQLite is a C library that implements an SQL database engine. It is a Relational Database Management System (or RDBMS). Most of the SQL databases work with the client/server model. Take MySQL for an example. To enter and receive data from a MySQL database you need to send a request to the MySQL server, which on reception will provide you with the appropriate response. Contrary to MySQL, SQLite databases are operated directly from the disk. There is no need to create requests to the server. One can create the main server script (server.js) using Express. Express.js is a very simple implementation for a web server, and provides the all CRUD (create, retrieve, update and delete) methods. Within computer programming, the acronym CRUD stands for create, read, update and delete. These are the four basic functions of persistent storage. Also, each letter in the acronym can refer to all functions executed in relational database applications and mapped to a standard HTTP method, SQL statement or DDS operation. Methods can also have the property of "idempotence" in that the side-effects of N > 0 identical requests is the same as for a single request. The methods GET, HEAD, PUT and DELETE share this property. CREATE

URI for POST request

( http://localhost:8000/api/customer" )

The POST method is used to request that the origin server accept the entity enclosed in the request as a new subordinate of the resource identified by the Request-URI in the Request-Line. New entities can be added using app.post method of express.js. The data of new entity is added in the form of json object. It is also checked with Postman library. The new object data becomes the part of database. A customer may use CRUD to create an account and access that account when returning to a particular site. The user may then update personal data or change billing information. On the other hand, an operations manager might create product records, then call them when needed or modify line items.

The data is posted in the form of json object in database through post method. the results shows the status success. It means the new object has created and added in database.



RETRIEVE

URI for Get request

( http://localhost:8000/api/customers" )

This shows all the users entites listed in the database. It provides the data in the form of json objects. The GET method means retrieve whatever information (in the form of an entity) is identified by the Request-URI. The method applied on this source is app.get which can be used with the express library.



URI for Get request of one entity

( http://localhost:8000/api/customer/id" )

This shows the user with the specific id mentioned in uri as json object. The method applied on this source is app.get which can be used with the express library.



UPDATE

URI for Delete request

( http://localhost:8000/api/customer/id" )

The PUT method requests that the enclosed entity be stored under the supplied Request-URI. If the Request-URI refers to an already existing resource, the enclosed entity SHOULD be considered as a modified version of the one residing on the origin server. If an existing resource is modified, either the 200 (OK) or 204 (No Content) response codes SHOULD be sent to indicate successful completion of the request. If the resource could not be created or modified with the Request-URI, an appropriate error response SHOULD be given that reflects the nature of the problem.

The data or object which one wants to update should be given in the form of json object in database through put method. PUT is HTTP 1.1 method to update data. The results shows the status success. It means the given object has modified and updated in database.

DELETE

URI for Delete request

( http://localhost:8000/api/customer/id" )

The DELETE method requests that the origin server delete the resource identified by the Request-URI. To delete any entity, enter id of that entity in uri. The specific entity will be delted from database. For this, express method app.delete is used.
